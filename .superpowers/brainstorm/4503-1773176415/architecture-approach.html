ACE NFT Marketplace — Architecture Approach

How should we structure this Vue.js + Tailwind v4 + Pinia project?

What we're building

An NFT marketplace dashboard with: sidebar navigation, NFT card grid with filters, user profile/balance panel, activity tabs, and a top creators leaderboard table. All styled to match the dark/light mixed theme from the reference design.

A

Single Page App (Recommended)

One Vue app with component-based architecture. Pinia stores for NFT data, user state, and UI filters. All sections as Vue components. Best for a dashboard-style page like this.

Pros

  • Natural fit for dashboard layout
  • Shared state via Pinia
  • Fast, no page reloads

Cons

  • Slightly more setup
B

Multi-Page with Vue Router

Full routing setup with separate pages for Dashboard, Collections, Settings, etc. Each sidebar link navigates to a distinct view.

Pros

  • Scalable for real app
  • Deep linking

Cons

  • Overkill for recreating one landing page
  • More boilerplate
C

Static HTML + Vue Islands

Mostly static HTML/Tailwind with small Vue components mounted for interactive parts (filters, tabs, balance widget).

Pros

  • Simplest setup
  • Fast to build

Cons

  • Doesn't fully leverage Vue/Pinia
  • Harder to maintain
